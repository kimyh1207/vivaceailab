# 1장. 개발 환경과 Git

## 들어가며

코드를 잘 짜는 것만큼 중요한 것이 있습니다. **코드를 안전하게 관리하고, 팀과 함께 일하는 방법**입니다.

아무리 뛰어난 개발자도 개발 환경이 엉켜 있으면 실력을 발휘할 수 없습니다. 반대로 개발 환경과 Git이 손에 익은 개발자는 어떤 팀에 들어가도, 어떤 프로젝트를 만나도 빠르게 적응합니다. 이것이 첫 번째 챕터에서 가장 먼저 이 주제를 다루는 이유입니다.

---

## 왜 개발 환경과 Git이 중요한가

> "내 컴퓨터에서는 됩니다."

개발자라면 한 번쯤 들어봤거나, 직접 해봤을 말입니다. 이 한 마디는 개발 환경이 통일되지 않았을 때 얼마나 많은 시간이 낭비되는지를 단적으로 보여줍니다.

개발 환경 설정은 **한 번 제대로 해두면 평생 자산**이 됩니다. 어떤 프레임워크를 쓰든, 어떤 언어를 배우든 개발 환경을 빠르게 세팅하는 능력은 변하지 않는 기본기입니다.

Git은 그보다 더 근본적입니다. Git 없이 협업은 불가능합니다. Git은 단순한 버전 관리 도구가 아닙니다. **팀의 대화 기록이자, 프로젝트의 역사이자, 실수를 되돌릴 수 있는 안전망**입니다. 커밋 하나하나가 쌓여 프로젝트가 어떻게 성장했는지를 보여주고, 브랜치 전략 하나가 팀 전체의 개발 속도를 결정합니다.
